    웹 컴포넌트 해외 판매기

    웹 개발을 하면서 가장 많이 사용하게 되는, 테이블 그리드, 차트 등과 같은 UI 컴포넌트 개발하여 해외 유수의 기업에 판매했던 경험을 들려드리고자 합니다. 흔하디 흔한 웹컴포넌트, 오픈소스로도 많이 나와 있는 컴포넌트들을 어떻게 해외 시장에 판매할수 있었을까요? 기업용 ERP(SAP) 시장에 최적화하였기 때문에 가능했습니다.

    지금은 많은 개발자들이 웹 프레임워크인 React, Vue, Angular 를 많이 사용하고 있는데, 제가 UI 컴포넌트를 판매를 하던 2014년도에는 위의 프레임워크가 알려지기 전입니다.
    이때 세계적인 ERP 솔루션 기업인 독일 SAP에서는 SAPUI5라는 웹 프레임워크를 발표하였고, 지금의 React, Vue, Angular 보다 훨씬 진화되었고, 앞서 있던 프레임워크였습니다. 참고로 SAPUI5는 JQuery를 개발했던 팀이 참여해서 개발이 되었습니다.

    십여년을 ERP 컨설턴트 및 개발자로 일했던 저는 SAPUI5의 부족한 기능을 개선하여 SAPUI5 기반의 다양한 웹 UI 컴포넌트를 개발하였고, 이를 지멘스, 존슨앤존스 같은 해외 유명 기업에 직접 판매를 하였습니다.
    SAPUI5의 부족한 기능을 개선한 제품이라는 의미로 SAPUI6라는 제품명으로 판매를 하였습니다.
    제품 홍보에서, 제품 판매 과정에서 필요했던 서류들, 그리고 해외 담당자와의 커뮤니케이션에서 발생했던 다양한 문제 등의 경험을 얘기드리고, 해외시장을 어떻게 개척하고 접근해야할지에 대해서 들려드리도록 하겠습니다.
